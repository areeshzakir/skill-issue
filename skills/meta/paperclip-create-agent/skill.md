Paperclip Create Agent walks you through the process of provisioning a new AI agent within the Paperclip ecosystem. It handles agent configuration including capability definition, permission scoping, tool access grants, workspace assignment, and governance policy attachment. The "hiring" metaphor reflects that each agent is created with a specific role, responsibilities, and boundaries, much like onboarding a new team member.

Use Paperclip Create Agent when you need a specialized agent for a new workstream, when workload requires parallel processing across multiple agents, or when you want to create a dedicated agent with restricted permissions for a sensitive task. It ensures every agent is created with proper governance guardrails from the start.

Example: Run Paperclip Create Agent to create a "Design Review Agent" with read access to the design repository, permission to use the critique and audit skills, a workspace scoped to the current project, and a governance policy that requires human approval before any file modifications. The agent is provisioned and ready to receive tasks within seconds.
