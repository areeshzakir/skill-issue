Onboard designs complete first-time user experiences including welcome screens, setup wizards, progressive onboarding tooltips, empty states, and guided feature tours. It creates flows that balance getting users to their "aha moment" quickly while collecting necessary setup information. Every onboarding element is designed to be skippable, resumable, and contextually relevant.

Use Onboard when launching a new product, adding a complex feature that requires explanation, or when analytics show high drop-off during initial user sessions. It is also valuable for redesigning empty states that currently show blank screens into invitations to take meaningful first actions.

Example: Your project management tool drops new users onto an empty dashboard. Run Onboard to design a 3-step setup wizard (create first project, invite a teammate, add first task), contextual tooltips that highlight key features on first encounter, and an empty dashboard state with a visual checklist of "getting started" milestones that transforms into the real dashboard as the user completes actions.
